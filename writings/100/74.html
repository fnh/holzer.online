I've argued before that I consider the page to be the most basic building block of a website, in the sense that the more specialized concepts like article, reply, blog post, note all have an is-a relationship to it.

That point of view shares a drawback of base abstractions that cover the commonalities a vast set of usecases: it is not very descriptive.

If so many things are pages, then what isn't? Arguably static assets like images, video, audio, stylesheets or scripts, and more conceptually also things like partials for common header/footer/navigation elements.
